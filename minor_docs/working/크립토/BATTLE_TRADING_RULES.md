# 원숭이 vs AI 로봇 — 매매 기준서

> 최종 작성일: 2026-03-20
> 코드: `lib/crypto/battle-trader.ts`
> 초기 자본: 각 $100

---

## 배틀 컨셉

밈코인 시장에서 **완전 랜덤 매매(원숭이)**와 **시그널 기반 매매(로봇)** 중 누가 이기는지 실제 가격 데이터로 검증하는 실험.

- 30분마다 크론 실행 (기존 크롤링 파이프라인 Phase 5)
- 실제 CoinGecko 가격 기준 PnL 계산
- 부분 청산 지원 (포지션 단위 관리)

---

## 원숭이 (Random Monkey)

### 철학
> "다트 던지기". 아무 코인이나 사서 아무 때나 판다.
> 밈코인 시장 자체의 수익률 벤치마크 역할.

### 진입 규칙

| 항목 | 규칙 |
|------|------|
| 판단 주기 | 30분마다 |
| 진입 확률 | **50%** (동전 던지기) |
| 코인 선택 | `crypto_coins` 활성 코인 중 **완전 랜덤** |
| 포지션 크기 | 포트폴리오의 **5~15%** (랜덤) |
| 최대 동시 포지션 | **5개** |

### 청산 규칙

| 조건 | 기준 |
|------|------|
| 홀드 기간 만료 | 진입 시 **1시간~7일** 중 랜덤 지정, 도달 시 자동 청산 |

- 손절/이익실현 없음
- 시그널 확인 없음
- 오직 시간만이 청산 조건

### 왜 이렇게 설계했나
- 원숭이가 너무 똑똑하면 벤치마크 의미 없음
- 홀드 기간이 랜덤이라 단타~스윙 모두 커버
- 밈코인 대박(10x)을 잡는 **운**도 반영됨
- 동시 5개 포지션으로 분산 효과

## AI 로봇 (Signal Robot)

### 철학
> "데이터가 말할 때만 산다. 데이터가 꺾이면 즉시 판다."
> 리스크 관리가 핵심. 잃지 않는 게 이기는 것.

### 진입 규칙

#### 필수 조건 (모두 충족해야 진입)

| 조건 | 기준 | 근거 |
|------|------|------|
| 시그널 라벨 | `strong_buy` 또는 `buy` (1h 윈도우) | 단기 모멘텀 확인 |
| weighted_score | **≥ 60** | 멘션 신뢰도 감쇠 반영된 종합 점수 |
| 멘션 수 | **≥ 5** (1h 내) | 최소 신뢰도 (1~2건은 노이즈) |
| 컨피던스 | **≥ 75** (아래 공식) | 복합 지표 충족 |

#### 컨피던스 점수 계산

```
confidence = 60 (기본)
  + 20  if mention_velocity > 1.0   (버즈 급증)
  + 15  if avg_sentiment > 0.3      (bullish 컨센서스)
  + 10  if fomo_avg > 0.5           (FOMO 감지)
─────
최대 105, 최소 충족 기준 = 75
```

→ 기본(60)만으로는 부족. **velocity, sentiment, fomo 중 최소 1개 이상** 보너스가 필요.

#### 포지션 관리

| 항목 | 규칙 | 근거 |
|------|------|------|
| 포지션 크기 | 포트폴리오의 **25%** | 2% 룰 역산: $2 손실 ÷ 8% = $25 |
| 최대 동시 포지션 | **3개** | 집중 투자, 최대 75% 투자 |
| 최소 현금 보유 | 포트폴리오의 **30%** | 기회 대기 + 리스크 버퍼 |
| 중복 진입 | 같은 코인 중복 불가 | 한 코인에 집중 위험 방지 |

### 청산 규칙

#### 손절 (Stop Loss)

| 단계 | 조건 | 액션 |
|------|------|------|
| TP 전 | 현재가 ≤ 진입가 × **0.92** (-8%) | **전량 청산** |
| TP1 후 | 현재가 ≤ **진입가** (본전) | **잔량 전부 청산** |

**왜 -8%인가:**

| 손실 | 회복에 필요한 수익률 |
|------|----------------------|
| -8% | **+8.7%** (관리 가능) |
| -15% | +17.6% |
| -20% | +25.0% |
| -50% | +100% |

→ -8%는 회복 가능한 최대 수준. 윌리엄 오닐 CAN SLIM 기법 참고.

#### 이익 실현 (Take Profit) — 3단계

| 단계 | 조건 | 액션 | 리스크:리워드 |
|------|------|------|---------------|
| **TP1** | PnL ≥ **+24%** | 포지션의 **1/3 청산** + 손절을 진입가로 이동 | 1:3 달성 |
| **TP2** | PnL ≥ **+50%** | 포지션의 **1/3 청산** | 이익 확보 |
| **TP3** | 시그널 반전 (아래 참조) | **나머지 전량 청산** | 대박 탑승 후 하차 |

**TP1 이후 손절이 진입가로 올라가는 이유:**
- 이미 1/3 수익 확보 → 나머지 포지션은 "공짜 게임"
- 최악의 경우에도 본전, 실질적으로 TP1 수익만 남음

#### 시그널 기반 청산 (TP3 트리거)

| 조건 | 기준 | 이유 |
|------|------|------|
| **시그널 반전** | 1h 라벨이 `sell` 또는 `strong_sell` | 모멘텀 소멸 |
| **센티먼트 급락** | 진입 대비 sentiment **-0.4 이상 하락** | 커뮤니티 분위기 전환 |
| **velocity 소멸** | mention_velocity **< 0.2** | 더 이상 화제가 아님 |

→ 이 3개 중 **하나라도** 해당되면 즉시 잔량 전부 청산.

## 비교 요약

| | 원숭이 | 로봇 |
|---|---|---|
| 진입 빈도 | 30분마다 50% | 30분마다 조건부 (하루 0~수건) |
| 코인 선택 | 랜덤 | 시그널 최고 점수 |
| 포지션 크기 | 5~15% 랜덤 | 25% 고정 |
| 최대 포지션 | 5개 | 3개 |
| 홀드 기간 | 1h~7d 랜덤 | 시그널 유지되는 한 무제한 |
| 손절 | 없음 | -8% (TP1 후 본전) |
| 이익 실현 | 없음 | +24% / +50% / 시그널 반전 |
| 현금 관리 | 없음 | 최소 30% 유지 |
| 예상 승률 | ~50% | >55% (시그널 엣지) |

## 핵심 원칙

### 2% 룰
한 거래에서 **전체 포트폴리오의 2% 이상 잃지 않는다.**
- 포트폴리오 $100 → 한 거래 최대 손실 = $2
- 포지션 25% ($25) × 손절 8% = $2 → **정확히 일치**

### 리스크:리워드 = 1:3
- 손절 -8% → 최소 목표 +24% (TP1)
- **3번 중 1번만 맞아도 본전**
- TP2(+50%), TP3(시그널 탑승)으로 추가 수익 가능

### 밈코인 특화
- 1h 윈도우 사용 (24h는 밈코인에겐 너무 느림)
- FOMO 지수를 컨피던스에 반영 (밈코인 펌프의 핵심 드라이버)
- velocity 소멸 감지 (버즈가 사라지면 가격도 따라감)
- 시간 제한 없음 (로봇) — 시그널이 살아있으면 3일이든 7일이든 홀드

## 데이터 흐름

```
30분 크론 → Phase 5 (battle)
  │
  ├─ 원숭이
  │   ├─ 오픈 포지션 확인 → hold_until 만료? → 청산
  │   └─ 50% 확률 → 랜덤 코인 매수 (5~15%, 1h~7d 홀드)
  │
  └─ 로봇
      ├─ 오픈 포지션 확인
      │   ├─ 현재가 vs 손절 라인 → -8% 이하? → 전량 청산
      │   ├─ TP1 (+24%) 도달? → 1/3 청산 + 손절→진입가
      │   ├─ TP2 (+50%) 도달? → 1/3 청산
      │   └─ 시그널 반전/센티먼트 급락/velocity 소멸? → 잔량 청산
      │
      └─ 시그널 평가
          ├─ 1h buy/strong_buy + score≥60 + mention≥5
          ├─ confidence ≥ 75 (velocity/sentiment/fomo 보너스)
          ├─ 현금 30% 이상 유지 가능?
          └─ 조건 충족 → 25% 매수 + 시그널 스냅샷 저장
```
